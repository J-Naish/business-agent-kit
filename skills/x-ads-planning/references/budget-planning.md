# Budget Planning

Use this reference to check whether the proposed X Ads structure can produce enough signal to be useful.

## Budget Principles

- Budget should follow the business outcome, not the number of available formats.
- Keep the launch structure narrow when the budget is small.
- Do not split by many audiences, keywords, geos, placements, and creatives unless each split has enough budget to learn.
- Judge budget sufficiency against the target action: CPA, CPI, CPL, purchase volume, qualified lead volume, or view quality.
- Premium placements and managed products require separate availability and minimum-spend checks with X.
- Do not hard-code universal CPM/CPC/CPA/CPV/CPI benchmarks. Use recent account data, current X forecasts, and market-specific external benchmarks as planning inputs.

## Signal Viability

| Goal | Minimum practical check |
|---|---|
| Website conversions / Sales | Can the budget generate enough conversion events to judge CPA/ROAS within the review window? |
| Lead generation | Can the budget generate enough qualified leads, not just form fills? |
| App installs | Is MMP tracking configured and can CPI support meaningful install volume? |
| App re-engagement | Is the retargetable app audience large enough and are downstream events tracked? |
| DPA | Are catalog events and product IDs matching reliably? |
| Awareness/video | Is reach/frequency large enough for a meaningful read? |

If the answer is no, narrow the structure or optimize to a higher-volume interim signal temporarily.

## Planning Math

Use these calculations before proposing structure:

| Question | Calculation |
|---|---|
| Can conversion optimization learn? | Monthly budget / target CPA = expected conversions |
| Is the campaign too fragmented? | Expected conversions / number of ad groups = signal per ad group |
| Is traffic volume enough? | Budget / expected CPC = clicks; clicks x landing-page CVR = conversions |
| Is app budget viable? | Budget / expected CPI = installs; installs x activation rate = meaningful events |
| Is video budget viable? | Budget / expected CPV = views; views x qualified-view rate = usable attention |

If expected conversions or qualified events are too low, consolidate ad groups, use a higher-volume interim event temporarily, or reduce the test scope.

## Benchmark Use

Benchmarks are useful only as rough order-of-magnitude checks.

| Metric | How to use it |
|---|---|
| CPM | Estimate reach and awareness viability; adjust for market, seasonality, brand safety, and premium inventory |
| CPC | Estimate traffic volume; adjust for objective, creative, audience, and landing-page relevance |
| CPV | Estimate video scale; adjust for video length, placement, and view definition |
| CPI | Estimate app-install volume; reconcile against MMP and downstream activation |
| CPA / ROAS | Use only after defining event quality, attribution window, margin, and new-vs-returning split |

When the user asks for budget viability, ask for or estimate a local benchmark range, show the math, and label the assumption. Do not present benchmark ranges as promises.

## Structure by Monthly Budget

These are planning ranges, not platform rules. Adjust for market CPM/CPC/CPA and category economics.

| Monthly budget | Practical structure |
|---|---|
| Small | 1 campaign, 1-2 ad groups, 2-4 distinct creatives. Prioritize one goal. |
| Medium | 2-3 campaigns or roles: prospecting, retargeting, creative/keyword test. |
| Large | Full-funnel structure with controlled testing, DPA/catalog, app or premium placements where justified. |
| Managed / launch burst | Separate media plan for Takeovers, Amplify Sponsorship, X Live, Branded Notifications, or Branded Hashtags. |

## Premium and Managed Product Cost Checks

Timeline Takeover, Spotlight Takeover, Amplify Sponsorship, X Live, Branded Notifications, and Branded Hashtags can have market-specific minimums, reservation rules, third-party tooling costs, or bundled media requirements. Do not encode a fixed minimum spend unless X provides a current quote for the exact market and date.

Ask X or the managed-service partner for:

- Minimum spend, package components, and whether supporting media is required.
- Reservation window, cancellation terms, and makegood policy.
- Creative approval deadlines and backup asset requirements.
- Reporting fields, lift-study availability, and post-campaign delivery timing.
- Third-party setup costs such as Arrow for Branded Notifications where applicable.

## Budget Split Starting Points

| Business model | Starting split |
|---|---|
| E-commerce | 60-80% prospecting/conversion, 10-25% retargeting/DPA, 10-20% creative tests |
| Lead gen | 50-70% prospecting/intent, 15-30% retargeting, 10-20% testing |
| App | 70-90% install or event acquisition, 10-30% re-engagement if audience exists |
| Brand | 70-90% reach/video/premium, 10-30% engagement or retargeting follow-up |
| Event | 40-60% pre-event, 20-40% live, 10-30% post-event follow-up |

Shift based on actual conversion quality, not only cheap platform outcomes.

## Change Discipline

- Avoid judging a conversion campaign before enough conversion delay has passed.
- Batch budget and bid changes; avoid changing budget, targeting, creative, and event definitions together.
- Increase budgets gradually unless the campaign is clearly underfunded or tied to a fixed event date.
- Keep a change log with date, campaign/ad group, change, reason, and expected observation window.
