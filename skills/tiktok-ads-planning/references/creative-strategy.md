# Creative Strategy

Use this reference when the user needs practical TikTok creative direction, not just specs. TikTok creative planning should define the concept, hook, proof, creator fit, CTA, and refresh system before asset dimensions.

## First Principles

- TikTok is a creative-signal platform. Targeting and bidding matter, but weak creative usually cannot be rescued by account structure.
- Treat creative supply as an operating constraint. If the team cannot produce new concepts regularly, plan a smaller structure and slower scale.
- Optimize for demand creation, not only demand capture. Strong TikTok creative often shows up as branded search, direct traffic, Amazon/search lift, Shop demand, and retargeting-pool growth.
- Speed matters, but there is a quality floor. Low-polish can work; unclear, misleading, or low-trust creative does not.

## Concept Before Variant

| Layer | What changes | Example |
|---|---|---|
| Concept | Persuasion logic | "Save time" vs "expert explains quality" vs "social proof" |
| Hook | First visual/line/motion | Comment rebuttal, demo first, problem callout |
| Proof | Why believe it | Demo, review, data, creator experience, comparison |
| Offer | Why act now | Bundle, trial, discount, quiz, consultation, Shop promo |
| Variant | Execution detail | Caption, crop, CTA, music, first frame |

A new caption is not a new concept if the same argument is doing the work.

## Hook Library

| Hook type | Best for | Pattern |
|---|---|---|
| Problem callout | Clear pain | "If [problem] keeps happening, try this..." |
| Demo first | Visual products/apps | Show use before explaining |
| Transformation tease | Visible before/after | Show outcome, then explain path |
| Negative comment rebuttal | Objection-heavy categories | Put skeptical comment on screen and answer it |
| Search-intent answer | Search Ads / education | "How to [solve problem] without [tradeoff]" |
| Social proof | Trust-sensitive offers | Review/comment first, then product proof |
| Founder/employee POV | Brand story, technical products | "We built this because..." |
| Pattern interrupt | Saturated feeds | Unexpected visual motion, framing, or scenario |

Avoid generic openers like "Have you ever wondered..." unless the next frame is unusually specific.

## Length and Pacing

| Length | Good fit |
|---|---|
| 6-15s | Simple products, app install, offer, retargeting |
| 15-30s | Demo, comparison, lead magnet, objection handling |
| 30-60s | Education, high-consideration products, proof sequence |
| 60s+ | Only when the content naturally earns attention and needs depth |

Set the persuasion job first, then use the shortest video that completes it. Current specs allow longer non-Spark videos, but most direct-response concepts still need fast proof.

## UGC, Creator, and Brand Creative

Creator-style creative often wins because it feels specific and believable, not because low production value is magic.

| Style | Good use | Watch-out |
|---|---|---|
| Creator/UGC | Trust, demo, social proof, native pacing | Over-scripted creators read as ads |
| Employee/founder | Expertise, story, quality explanation | Can become self-centered |
| Brand-shot native | Premium categories, compliance, controlled claims | Must still use TikTok pacing |
| Polished brand film | Launches and managed premium formats | Often weak for auction conversion |
| Symphony/AI-assisted | Variants, b-roll, localization, refresh | Generic AI output can homogenize creative |

## Creator Brief Checklist

A good creator brief includes:

- Audience and moment of use.
- Problem, misconception, or desire to dramatize.
- Required proof or demo.
- Claims to use and claims to avoid.
- Hook options, not a rigid script.
- CTA and destination.
- Usage rights, Spark authorization duration, account-linking needs.
- Disclosure and regulated-category requirements.

Control claims and compliance; let the creator control native delivery.

## Fatigue Diagnosis

| Symptom | Likely meaning | Action |
|---|---|---|
| CTR declines | Hook or audience fit weakening | Add new hooks / first frames |
| 2s or 6s view rates decline | Opening is losing attention | Replace first 1-3 seconds |
| Hold rate drops while CTR holds | Body does not pay off the hook | Rewrite demo/proof section |
| CPM rises while engagement weakens | Auction quality or audience response is degrading | Refresh creative, broaden safely, review comments |
| Comments become repetitive/negative | Message fatigue or claim issue | Replace angle, address objection, check policy |
| CPA rises with frequency | Audience saturation | Add concepts or expand audience/placement |

Refresh the hook first when the body still converts. Replace the concept when multiple hook refreshes fail.

## Comment Mining

Use comments as qualitative research:

- Objections become rebuttal hooks.
- Repeated questions become Search Ads and educational scripts.
- Praise reveals proof language.
- Complaints reveal product/landing-page issues.
- Confusion reveals messaging mismatch.

Do not chase comments as an outcome unless engagement/community is the goal.

## Practices to Avoid

- "UGC always wins." Better: credible, specific, native creative usually wins.
- "Shorter is always better." Better: as short as possible while completing the persuasion job.
- "High CTR means good creative." Better: pair CTR with hold rate, CVR, comment quality, and downstream quality.
- "AI solves fatigue." Better: AI can increase throughput, but human hooks and differentiated concepts still matter.
