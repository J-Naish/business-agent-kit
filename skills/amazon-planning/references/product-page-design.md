# Product Detail Page Design

Use this reference to design Amazon product detail pages, including title, bullet points, description, search terms, A+ Content, variations, and Amazon SEO. Requirements vary by category and marketplace; verify current style guides in the target account.

## Required and Recommended Fields

| Field | Role |
|---|---|
| Product identifier | GTIN such as UPC, EAN, ISBN, or JAN, unless exempt |
| Title | Primary searchable product name and shopper scan line |
| Brand | Brand/manufacturer name; should align with Brand Registry where relevant |
| Main image | Required product image; must comply with Amazon image policy |
| Price | Selling price, including local tax display rules where applicable |
| Inventory | Available quantity or FBA inbound plan |
| Condition | New, used, renewed, collectible, or category-specific condition |
| Category/browse node | Determines discoverability, style guide, and eligibility |
| Bullet points | Key benefits, features, use cases, compatibility, and objections |
| Product description | Additional context, story, instructions, or technical detail |
| Search terms | Backend search keywords within Amazon's current limits |
| Dimensions/weight | Important for shopper confidence and fulfillment fees |
| Compliance attributes | Material, ingredients, safety, warnings, certifications, age grade, country-specific fields |

## Title Strategy

The title must balance search relevance, shopper clarity, and category compliance.

### General Pattern

```text
Brand + Product Type + Primary Differentiator + Key Attribute/Size/Quantity + Use Case or Compatibility
```

Examples:

```text
ProWidget Stainless Steel Water Bottle, Insulated, 32 oz, Leakproof Lid, For Travel and Gym
ProWidget USB-C Laptop Stand, Adjustable Aluminum Riser, Foldable Desk Stand for 13-16 in Laptops
```

### Title Rules

- Put the most important query and product type early.
- Avoid keyword stuffing, repeated words, promotional claims, emojis, and unsupported superlatives.
- Follow category-specific maximum length and formatting rules.
- Use shopper language, not only internal product names.
- Include size, quantity, compatibility, or material when it changes purchase fit.

## Bullet Points

Use bullet points to convert, not just describe.

Recommended structure:

| Bullet | Role |
|---|---|
| 1 | Main value proposition and primary use case |
| 2 | Differentiating feature with proof |
| 3 | Fit, size, compatibility, or what is included |
| 4 | Usage, care, safety, or objection handling |
| 5 | Brand trust, warranty, sustainability, or support |

Pattern:

```text
Benefit-led heading: feature proof -> shopper outcome.
```

Avoid:

- Claims that cannot be substantiated.
- Medical, health, or regulated claims without approval.
- Competitor attacks.
- Repeating the title.
- Stuffing every keyword into every bullet.

## Product Description

Use the product description for:

- Product story and deeper explanation.
- Setup, care, or usage instructions.
- Compatibility details.
- Materials and manufacturing process.
- Brand voice when A+ Content is unavailable.

If A+ Content is available, let the description support details that A+ does not cover.

## Backend Search Terms

Use backend search terms for relevant synonyms and alternate phrasing that do not fit naturally in visible copy.

Guidelines:

- Do not repeat words already covered heavily in title/bullets unless needed.
- Include synonyms, spelling variants, and use-case language.
- Avoid competitor trademarks unless policy and local law allow it.
- Do not include irrelevant high-volume terms.
- Respect current byte/character limits in the target marketplace.

## A+ Content

Use A+ Content to increase conversion and brand trust.

Common module strategy:

| Module | Role |
|---|---|
| Hero | Brand/product promise and visual context |
| Feature blocks | Main benefits with proof |
| Comparison chart | Help shoppers choose the right SKU |
| Lifestyle | Show usage and emotional payoff |
| Technical/spec | Reduce compatibility and fit uncertainty |
| Brand story | Build trust when brand matters |

## Variations

Use parent-child variations only when products are true variations under Amazon's category rules, such as size, color, flavor, quantity, or style.

Do not merge unrelated products to pool reviews. Variation misuse can cause suppressed listings or account risk.

## Amazon SEO

Amazon search performance is influenced by relevance and performance:

| Driver | Examples |
|---|---|
| Relevance | Title, bullet points, search terms, category, attributes, browse node |
| Performance | Sales velocity, conversion rate, price, availability, reviews, fulfillment, ad traffic quality |
| Shopper trust | Ratings, reviews, images, A+ Content, delivery promise, return confidence |

Treat ads and SEO as a loop. Ads reveal converting queries; listings should absorb those learnings into visible copy and backend terms where appropriate.

## Product Detail Page Audit

Before advertising, check:

- Main image wins the click in search results.
- Title is readable and includes the core product query.
- First three bullet points address the largest purchase drivers.
- Secondary images answer common objections.
- Price and delivery are competitive.
- Review count and rating are not fatal relative to competitors.
- Variations are clear and compliant.
- A+ Content or description supports the ad promise.
- Inventory can support expected demand.
