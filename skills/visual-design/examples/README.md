# Examples Index

Each file is a detailed design system modeled on a real brand — color palette, typography, components, depth, layout rules, and prompt-ready snippets. Use them three ways:

1. **Reference for `DESIGN.md` format and depth** — see how a complete design system is documented.
2. **Pattern source** — borrow color strategy, type hierarchy, shadow systems.
3. **Starting point** — if a brand below is close to the target tone, adapt it instead of starting from zero.

Scan this index first to find 1–2 examples whose tone matches the brief, then read those full files.

## How to pick

- **Match the medium first** — a slide deck for an enterprise audience pulls from Stripe / HashiCorp / Notion, not PlayStation.
- **Match the energy** — premium-quiet (Apple/Notion), warm-editorial (Claude/WIRED), developer-precise (Vercel/Linear), bold-expressive (PlayStation/Spotify).
- **Then adapt** — never copy 1:1. Use 1–2 examples as a tonal anchor and synthesize.
